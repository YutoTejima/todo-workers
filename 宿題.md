# 異なるユーザーのタスクを消せてしまうことの確認手順と証明

## 原因

タスクをいじる際に認証のミドルウェアしかないため、ログインしているかいないかの区別しかできておらず、現在ログインしているユーザーが「該当タスクを作成したユーザーかどうかの判別」ができていない

## 証明

事前にユーザー A でタスクを作成しておく

1. ユーザー B を作成
2. ユーザー B でログイン
3. ユーザー A で作成したタスクを削除する

## 解決策

ミドルウェアでセットされたセッションからログイン中のユーザーの userId を取り出しておく roginUserId: userId
各 API で条件に where: { userId: roginUserId } を追加する
